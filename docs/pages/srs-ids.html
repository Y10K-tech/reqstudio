SRS IDs

SRS-IDs encode project, area, type, and a sequence number to enable traceability across documents, commits, and branches. The app detects SRS-IDs in content and exposes them in the UI and commit helpers.

Format

Y10K-<PROJ>-<AREA>-(HL|LL|CMP|API|DB|TST)-<NNN>
  • PROJ: Project code (e.g., PAY, CORE).
  • AREA: Subsystem/domain (e.g., API, WEB, DB).
  • TYPE: One of HL, LL, CMP, API, DB, TST.
  • NNN: Zero-padded integer sequence (e.g., 001, 042).

Examples

Y10K-PAY-API-HL-001
Y10K-PAY-API-LL-012
Y10K-COR-WEB-CMP-107
Y10K-OPS-DB-ADR-004   (ADR IDs exist as document titles; commit messages can still include them)

Usage Guidelines

  • Include the ID at the top of the document and in commit messages.
  • Prefer one primary SRS-ID per document; link related IDs in a references section.
  • Use branch names like feature/Y10K-PAY-API-LL-012-short for cross-linking.
  • Consistent casing is recommended; detection is generally case-insensitive.
