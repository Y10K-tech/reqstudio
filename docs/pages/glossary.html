Glossary

  • SRS — Software Requirements Specification.
  • SRS-ID — An identifier used to link requirements to commits, branches, and artifacts.
  • ADR — Architecture Decision Record documenting a decision, its context, and consequences.
  • HL — High-level specification.
  • LL — Low-level specification.
  • CMP — Component specification.
  • MCP — Model Context Protocol for tool integration in LLM workflows.
  • GitFacade — Wrapper around GitPython providing app-friendly operations.
