Git Backend

core/git_backend.py wraps GitPython in a small, UI-friendly API. It centralizes error handling, string formatting, and safe operations to keep the UI code clean.

Errors

Errors are raised as GitError with user-friendly messages (e.g., repo not open, no remote configured).

Operations

open(root: Path)
init(root: Path)
is_open() -> bool
current_branch() -> str
relpath(abspath: str|Path) -> str
commit(paths?: List[Path], message: str = "")
list_branches() -> List[str]
create_branch(name: str, checkout: bool = True)
checkout(name: str)
log_file(relpath: str, max_count: int = 100) -> List[CommitMeta]
get_file_at_commit(commit_hash: str, relpath: str) -> str
push() -> str | None
pull() -> str | None

History Data

Each commit in log_file returns { hash, short, author, when, msg } optimized for list rendering.

Notes

  • Empty commits are blocked with a clear message.
  • Detached HEAD is shown as (detached) instead of throwing.
  • Remote operations require a configured remote; otherwise a descriptive error is raised.
