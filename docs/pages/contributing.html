Contributing

Contributions are welcome. Please open issues for bugs or proposals and submit PRs with focused changes. Keep the code style consistent with existing modules and update documentation when behavior changes.

Setup for Development

python -m venv .venv
. .venv/bin/activate                 # Windows: .venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install .

# Optional (if defined):
# python -m pip install .[dev]

Guidelines

  • Prefer small, focused PRs with a clear rationale.
  • Add tests when patterns exist; avoid introducing new frameworks without discussion.
  • Keep UI responsive and code straightforward; avoid overengineering.
  • Document user-facing changes in this docs site.
