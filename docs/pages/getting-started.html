Quickstart

Open or initialize a Git-backed workspace, write requirements, commit with SRS-IDs, and export PDFs.

1) Launch and Pick a Workspace

  • Start the app with python app.py.
  • Use the menu to open an existing folder or initialize a new Git repo.
  • The status bar shows the current workspace and branch.

2) Create or Open a Document

  • Work with Markdown or plain text files (.md, .txt).
  • Use built-in templates (HL/LL/CMP/ADR/Ticket) to accelerate drafting.
  • The editor provides light Markdown highlighting and SRS-ID highlighting.

3) Traceability with SRS-IDs

  • Include SRS-IDs in your document content, e.g. Y10K-PAY-API-LL-012.
  • The app detects SRS-IDs and surfaces them for commit messages and branch naming.

4) Commit and Branch

  • Stage and commit either the current file or all changes.
  • Create or switch branches; push/pull if a remote is configured.
  • View per-file history and inspect earlier versions inline.

5) Export

  • Export PDFs via the Qt print pipeline for review/approval.
  • For styled exports, see roadmap (pandoc/WeasyPrint integration planned).
