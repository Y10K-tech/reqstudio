Overview

Y10K ReqStudio is a Git-driven editor for software requirements and SRS workflows. It focuses on speed, traceability, and auditability by keeping your specifications as plain text/Markdown inside a Git repository. The desktop app is built with Python and PyQt6, and integrates directly with Git for branching, staging, committing, and per-file history with inline diffs. PDF export and built-in SRS templates help you move from drafting to approvals efficiently. This documentation covers concepts, UI, CLI/API, and the code architecture.

Key Features

  • Git-backed workspace and file history with diffs
  • Markdown/Text editing with lightweight highlighting
  • SRS-ID detection for traceability
  • Built-in templates: HL, LL, CMP, ADR, Ticket (Swedish and English)
  • PDF export via Qt print pipeline
  • Roadmap for richer PDF theming, validation, and plugins
  • Planned interfaces: CLI, FastAPI server, MCP integration

Who Is It For?

  • Engineers and teams who prefer plain text, Git, and code review for requirements and SRS.
  • Organizations that need traceability, branch-based workflows, and simple exports for approvals.
  • Projects that want to integrate SRS work into the wider dev toolchain (CI, repos, PRs).
